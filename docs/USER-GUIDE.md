## 1. インストール方法

### 前提条件

- **VS Code** バージョン 1.80 以上
- **OS**: Windows 10 / 11

### 手順（VSIXファイルからインストール）

1. 共有フォルダから **`md-visual-editor-<バージョン>.vsix`** をローカルにコピー
   > 共有先パスは管理者から案内されます
2. VS Code を開く
3. `Ctrl + Shift + P` でコマンドパレットを開く
4. **「Extensions: Install from VSIX...」** と入力して選択
5. コピーした `.vsix` ファイルを選択
6. インストール完了後、**VS Code を再起動**

> コマンドラインからもインストールできます:
> ```powershell
> code --install-extension md-visual-editor-0.5.6.vsix
> ```

### インストール確認

1. VS Code で任意の `.md` ファイルを開く
2. ファイルを右クリック → **「エディターを開く方法」** → **「Markdown Visual Editor」** を選択
3. プレビュー形式の編集画面が表示されれば成功

## 2. Markdownファイルの開き方

### 方法A: 右クリックメニューから開く（推奨）

1. エクスプローラーで `.md` ファイルを右クリック
2. 「Open With...」（別のエディターで開く）を選択
3. 「Markdown Visual Editor」を選択

### 方法B: 既に開いているファイルを切り替える

1. `.md` ファイルを通常のテキストエディタで開く
2. エディタタブの右上にある「…」メニューをクリック
3. 「Reopen Editor With...」を選択
4. 「Markdown Visual Editor」を選択

### 方法C: コマンドパレットから開く

1. `.md` ファイルを開いた状態で `Ctrl + Shift + P`
2. 「Reopen With...」と入力
3. 「Markdown Visual Editor」を選択

### 方法D: タイトルバーのボタン／コマンドから開く（v0.5.3 追加）

`.md` ファイルを**標準のテキストエディタ**で開いているときは、ワンクリックでビジュアルエディタに切り替えられます。

- エディタタブ右上（タイトルバー）に表示される **「Markdown ビジュアルエディタで開く」** ボタン（プレビューアイコン）をクリック
- または `Ctrl + Shift + P` でコマンドパレットを開き、**「Markdown Visual Editor: Markdown ビジュアルエディタで開く」** を実行

このボタン／コマンドは `.md` ファイルをテキストエディタで開いているときだけ表示されます。逆に、ビジュアルエディタから元のテキスト表示に戻したい場合は、ツールバー右端の **📝 テキストで開く** ボタンを使います（[6. ツールバーの使い方](#6-ツールバーの使い方) 参照）。この2つを使うことで、テキスト編集とビジュアル編集を自由に行き来できます。

## 3. 基本操作 — ブロックという考え方

Markdown Visual Editor では、文書は **「ブロック」** という単位に分割され、それぞれが1つの編集可能な領域として表示されます。ブロックには2種類あります（v0.5.4 以降の仕様）。

### テキストセクション（見出し単位でまとまったブロック）

**H1（`#`）または H2（`##`）見出しから、次の H1/H2 見出しの直前まで** が、まとめて **1つのブロック** になります。見出し配下の H3〜H6 の小見出し、段落、リスト、引用、区切り線（`---`）などは、すべて同じセクションの中に含まれます。

> 例えば以下の Markdown は、`## 概要` から次の `## 詳細` の直前までがまとめて1ブロックとして編集されます。
> ```markdown
> ## 概要
> ここに説明文。
> - 箇条書き1
> - 箇条書き2
> ### 補足
> 補足の段落。
>
> ## 詳細
> ...
> ```

見出しの付いていない先頭の段落なども1つのセクションとして扱われます。

### 特殊ブロック（単独で1ブロックになる要素）

以下の要素は、周囲のテキストとはまとめられず、**必ず単独で1つのブロック**になります。

- テーブル
- コードブロック（&#96;&#96;&#96; で囲んだ部分。プレーンなコードだけでなく、Mermaid ダイアグラムや数式ブロックもここに含まれます）

つまり「見出しの下に説明文があり、その下に表がある」という文書では、「見出し＋説明文」で1ブロック、「表」で別の1ブロックというように分かれます。

### なぜこの仕組みを知っておくべきか

- 1つの見出しセクションをまとめて編集するので、**関連する内容を続けて書き換えやすくなります**。
- 逆に、**見出しの単位を変えずに文章の一部だけ独立したブロックにしたい場合は、H1/H2 の見出しを追加してセクションを分割してください**。
- テーブル・コードブロック・Mermaid・数式は常に独立したブロックのままなので、他のテキストと混ざる心配はありません。

## 4. ブロックの選択・編集・追加

### 4.1 ブロックの選択

- ブロックを **クリック** → そのブロックだけを選択（枠線で表示）
- `Ctrl + クリック` → 選択のオン/オフを切り替え（複数選択）
- `Shift + クリック` → 範囲選択
- 何もない背景をクリック → 選択解除

### 4.2 編集の開始と確定

1. **編集したいブロックをダブルクリック**（または、ブロックを選択してから `Enter` / `F2`）
   → テキストセクションの場合はテキストエリアに切り替わり、Markdown ソースが表示されます
2. **テキストを編集** → Markdown 記法で直接編集（`**太字**`、`*斜体*`、`- リスト` 等）
3. **確定** → 以下のいずれかで確定:
   - `Escape` キー（変更がある場合は破棄確認）
   - `Ctrl + Enter`
   - ブロックの外側をクリック

編集内容は即座にレンダリングされ、ファイルに自動保存されます。

> **編集後はブロックにフォーカスが戻ります（v0.5.3）。** `Escape` または `Ctrl + Enter` で確定すると、そのブロックが選択された状態に戻るため、続けて `↑` / `↓` キーで別のブロックに移動したり、`Enter` で再編集したりできます。詳しくは [5. キーボードだけで編集する](#5-キーボードだけで編集する) を参照してください。

### 4.3 ブロックの追加（右クリックメニュー、v0.5.2 追加）

新しいブロックは、既存のブロックを右クリックして挿入できます。

1. 基準にしたいブロックを右クリック
2. コンテキストメニューから以下のいずれかを選択:
   - **「⬆ 上にブロックを追加…」** — このブロックの直前に挿入
   - **「⬇ 下にブロックを追加…」** — このブロックの直後に挿入
3. サブメニューから挿入したい要素の種類を選択:

   | 記号 | 種類 |
   |---|---|
   | ¶ | 段落 |
   | H1 / H2 / H3 | 見出し |
   | • | 箇条書きリスト |
   | 1. | 番号付きリスト |
   | ☑ | タスクリスト |
   | ⊞ | 表 |
   | { } | コードブロック |
   | ∑ | 数式 |
   | ❝ | 引用 |
   | ─ | 水平線 |
   | ◇ | Mermaid ダイアグラム…（種別ピッカーが開きます） |

4. 選択すると、その場に新しいブロックが挿入され、**そのまま編集状態に入ります**（Mermaid の場合はダイアグラム種別ピッカーが開きます）。

> ブロックを選択した状態であれば、右クリックせずにキーボードだけでも同じ操作ができます。`Ctrl + Enter` で「下に追加」メニュー、`Ctrl + Shift + Enter` で「上に追加」メニューが開きます。

### 4.4 その他のブロック操作（右クリックメニュー）

ブロックを右クリックすると、選択状態に応じて以下のメニューが表示されます。

**単一ブロック選択時**

`✎ 編集` / `⬆ 上にブロックを追加…` / `⬇ 下にブロックを追加…` / `✂ 切り取り` / `⧉ コピー` / `📋 貼り付け(このブロックの後ろ)` / `↑ 上に移動` / `↓ 下に移動` / `📄 PDF として出力` / `🗑 削除`

**複数ブロック選択時（2件以上）**

`✂ 切り取り(N件)` / `⧉ コピー(N件)` / `📋 貼り付け` / `📄 PDF として出力` / `🗑 N件のブロックを削除`

**背景（ブロック外）を右クリックした場合**

`📋 貼り付け(末尾に追加)` / `📄 PDF として出力`

その他、ブロックはドラッグ＆ドロップでも並べ替えられます。ブロックにマウスを乗せると表示されるハンドル（`⋮⋮`）、またはブロック自体をドラッグし、上下の中点を基準にした線が表示された位置にドロップします。複数選択中はまとめて移動します。

削除時は確認ダイアログが表示され、削除前の内容は自動的にクリップボードへコピーされます。

## 5. キーボードだけで編集する（v0.5.3）

マウスを使わず、キーボードだけで一通りの編集ができます。ブロックからブロックへ移動しながら連続して編集する場合に特に便利です。

### ブロックを選択している状態（編集はしていない）

| キー | 動作 |
|---|---|
| `Enter` / `F2` | 編集を開始 |
| `Delete` | 削除（確認ダイアログあり。削除前にクリップボードへコピー） |
| `↑` / `↓` | 前後のブロックへフォーカス移動＋単一選択 |
| `Ctrl + Enter` | 「下にブロックを追加」メニューを開く |
| `Ctrl + Shift + Enter` | 「上にブロックを追加」メニューを開く |
| `Tab` | 次のブロックへフォーカス移動（ブラウザ標準のフォーカス移動） |

### テキストセクションを編集している状態（テキストエリア）

| キー | 動作 |
|---|---|
| `Escape` | 変更があれば破棄確認のうえ編集を完了。**完了後はそのブロックにフォーカス＋選択が戻ります** |
| `Ctrl + Enter` | 変更を確定（同上、フォーカスが戻ります） |
| `Alt + ↑` / `Alt + ↓` | 変更を確定し、**そのまま前後のブロックの編集へ直接ジャンプ**（選択に戻らず即編集） |
| `Ctrl + B` | 太字の書式を適用 |
| `Ctrl + I` | 斜体の書式を適用 |
| `Tab` | 半角スペース4個を挿入 |

`Alt + ↑` / `Alt + ↓` を使うと、複数のブロックを次々に編集していく作業を、マウスに触れずに行えます。

### 複数のブロックを選択している場合（2個以上選択時）

`Ctrl + C` / `Ctrl + X` / `Ctrl + V`（コピー／切り取り／貼り付け）、`Delete` または `Backspace`（**2個以上選択しているときのみ**削除）、`Escape`（選択解除）

### グローバル（どこにいても使える）

| キー | 動作 |
|---|---|
| `Ctrl + F` | 検索バーを表示 |
| `Ctrl + H` | 検索／置換バーを表示 |
| `Ctrl + Z` | 元に戻す（Undo） |
| `Ctrl + Shift + Z` / `Ctrl + Y` | やり直す（Redo） |

> テキスト入力中（テキストエリアなど）は、Undo/Redo はブラウザ標準の動作に委ねられます。また `Ctrl + S` はこの拡張機能では横取りされず、VS Code 標準の保存操作がそのまま使えます。

## 6. ツールバーの使い方

画面上部のツールバーから各種要素を挿入できます。

### 書式ボタン

| ボタン | 機能 | 挿入される Markdown |
|---|---|---|
| **B** | 太字 | `**太字テキスト**` |
| *I* | 斜体 | `*斜体テキスト*` |
| ~~S~~ | 取り消し線 | `~~取り消しテキスト~~` |
| `</>` | インラインコード | `` `コード` `` |

### 見出しボタン

| ボタン | 機能 |
|---|---|
| H1 | `# 見出し1` を挿入 |
| H2 | `## 見出し2` を挿入 |
| H3 | `### 見出し3` を挿入 |

### 構造要素ボタン

| ボタン | 機能 |
|---|---|
| • List | 箇条書きリストを挿入 |
| 1. List | 番号付きリストを挿入 |
| 🔗 | リンクを挿入 |
| ⊞ | テーブルテンプレートを挿入 |
| { } | コードブロックを挿入 |
| ∑ | 数式ブロックを挿入 |
| ◇ Mermaid | ダイアグラム種別選択ピッカーを表示（20 種類から選択） |

### ツールバー右端のボタン

| ボタン | 機能 |
|---|---|
| ●/○ 保存インジケータ | ドキュメントの未保存（dirty）状態を表示するのみ（クリック不可） |
| ↶ / ↷ | Undo / Redo |
| 🔍 検索 | **検索／置換バー**を表示。`Ctrl + F` / `Ctrl + H` でも起動でき、大文字区別・正規表現・一括置換に対応 |
| ☀️ / 🌙 テーマ切替 | エディタと Mermaid 図のテーマを **自動（VS Code に追従） → ライト強制 → ダーク強制** と循環。選択は保存される |
| 📝 テキストで開く | 現在の Markdown を **標準テキストエディタ** で開き直し。逆方向（テキスト→ビジュアル）は [2. 方法D](#2-markdownファイルの開き方) を参照 |

### 動作ルール

- **編集中のブロックがある場合**: カーソル位置に挿入
- **編集中のブロックがない場合**: **挿入位置ピッカー**が表示され、「先頭に挿入」または任意のブロックの後に挿入位置を選択できます。ブロックを選択した状態でピッカーを開くと、そのブロックの位置があらかじめ選択されており、`Enter` で即確定できます

> **Mermaid ボタンの場合:** 編集状態に関係なく、まずダイアグラムの種類（20 種類から選択）を選んだ上で、挿入位置ピッカーで位置を指定します。

### リンクのクリック動作

- **リンクは通常のクリックでそのまま開きます。修飾キー（`Ctrl` や `Cmd`）は不要です。**
  - `http://` / `https://` / `mailto:` は OS のデフォルトブラウザ／メーラーで開きます。
  - それ以外のスキームは VS Code 内で開きます。
  - 相対パスのリンクは、開いている `.md` ファイルの場所を基準に解決されます。
  - `#` だけのリンク（ページ内アンカー）はクリックしても何も起こりません。

## 7. テーブルの編集

Markdown のテーブルは、そのブロックの中で直接セルを編集する GUI で操作します。

### 操作手順

1. テーブルのブロックを選択し、編集を開始（ダブルクリック、または選択して `Enter` / `F2`）
2. 各セルが**自動で高さが伸び縮みするテキストボックス**になります。クリックして直接入力してください（長い文章は自動的に折り返されます）
3. セル内で改行すると、保存時に Markdown の `<br>` に変換されます（逆に `<br>` を含むセルを開くと、セル内では改行として表示されます）
4. 列・行の追加や削除は、セルを **右クリック** して以下のメニューから行います:
   - 列を追加（左 / 右）
   - 行を追加（上 / 下）
   - この列を削除（列が2列以上ある場合のみ）
   - この行を削除（行が2行以上ある場合のみ）
5. 列の境界線（`.col-resize-handle`）をドラッグすると列幅を調整できます。**ただしこれは表示上の調整のみで、Markdown ファイルには保存されません**（再度開くと元の幅に戻ります）

### 制限事項

- **セルの配置（左寄せ・中央・右寄せ、`:---:` などの指定）は GUI では設定できません。** さらに、**すでに配置指定がある表をビジュアルエディタで開いて保存すると、その配置指定は失われます**（区切り行が `---` だけの状態で書き出されます）。配置を厳密に保ちたい表は、テキストエディタ側で直接編集することを検討してください。
- テーブル編集には専用の Undo はありません（`Ctrl + Z` はドキュメント全体の Undo として働きます）。

## 8. 画像の挿入

画像ファイルをドラッグ＆ドロップするだけで、Markdown に画像参照を挿入できます。

### 操作手順

1. エクスプローラーなどから画像ファイルを、**既存のブロックの上** または **エディタの余白（背景）** にドラッグ＆ドロップ
2. 画像は自動的に、開いている `.md` ファイルと同じ階層の **`images/` フォルダ** に保存されます（フォルダが無ければ作成されます。同名ファイルがある場合は名前が自動的に調整されます）
3. カーソル位置（またはドロップした場所）に `![alt](images/ファイル名)` の形式で画像参照が挿入されます

### 対応する拡張子

`png` / `jpg` / `jpeg` / `gif` / `webp` / `svg` / `bmp` / `ico` / `avif`

### 表示について

- 相対パスの画像はプレビュー上で自動的に解決されて表示されます
- 読み込み中はプレースホルダ、読み込みに失敗した場合は破線の枠で表示されます

## 9. 数式の編集

インライン数式（`$...$`）とディスプレイ数式（`$$...$$`）、および &#96;&#96;&#96;math フェンスで囲んだ数式ブロックが KaTeX でレンダリングされます。

### 数式ブロックの編集

1. &#96;&#96;&#96;math ブロックにマウスを移動 → 「✎ 数式を編集」ボタンが表示されます
2. クリックすると数式エディタが開きます。左側がテキストエリア（LaTeX 入力）、下部または右側に**ライブプレビュー**が表示され、入力に応じて（約200ms のデバウンス後に）自動更新されます
3. 画面には **LaTeX 記号パレット** が用意されており、クリックするだけでよく使う記号を挿入できます。パレットは以下の7グループに分かれています:
   - 構造（分数・累乗・添字・根号など）
   - 演算子
   - 関係（等号・不等号など）
   - 大記号（総和・積分など）
   - ギリシャ文字
   - 集合・論理記号
   - 行列・整列
4. `Ctrl + Enter` で保存、`Escape` でキャンセル（変更がある場合は確認）。`Tab` は半角スペース4個を挿入します

### インライン数式・ディスプレイ数式

本文中に `$x^2 + y^2 = 1$` のように書くと、そのままレンダリングされます。コードブロックやリンクの中、既に数式として描画済みの部分には反映されません。

数式にエラーがある場合は、赤みがかった背景でエラー内容が表示されます（マウスを乗せると理由がツールチップで確認できます）。

## 10. Mermaidダイアグラムの編集（概要）

Mermaid コードブロック（&#96;&#96;&#96;mermaid ～ &#96;&#96;&#96;）にマウスを移動すると「✎ ダイアグラムを編集」ボタンが表示されます。

ダイアグラムの種類に応じて、**ビジュアルエディタ（GUI）** または **コードエディタ** が起動します。

### 対応一覧

| 種類 | エディタ | Mermaid 先頭行 |
|---|---|---|
| フローチャート | **ビジュアル（GUI）** | `graph TD` / `flowchart LR` 等 |
| クラス図 | **ビジュアル（GUI）** | `classDiagram` |
| シーケンス図 | **ビジュアル（GUI）** | `sequenceDiagram` |
| マインドマップ | **ビジュアル（GUI）** | `mindmap` |
| 象限チャート | **ビジュアル（GUI）** | `quadrantChart` |
| ガントチャート | **ビジュアル（GUI）** | `gantt` |
| ER 図 | **ビジュアル（GUI）** | `erDiagram` |
| その他すべて | コード + プレビュー（一部は簡易フォーム編集） | 上記以外の Mermaid 構文 |

上記7種類は、ドラッグ操作などを含む特に高機能な専用ビジュアルエディタです。それ以外の主要な図（状態遷移図・パイチャートなど多数）も簡易的なフォーム形式の GUI で編集できるものがありますが、対応していないものやコード編集のみのもの（ZenUML）もあります。詳しくは [18. その他のMermaidダイアグラム（コードエディタ）](#18-その他のmermaidダイアグラムコードエディタ) を参照してください。

### Mermaid 図の拡大・縮小（ズーム／パン）

図が画面に収まらない場合のズーム／パン操作は、**表示される場所によって操作方法が少し異なります。**

**a. 本文プレビュー内の Mermaid 図（v0.5.5 追加）**

ドキュメントの本文中に表示されている Mermaid 図（種類を問わず）にマウスを乗せると、操作ボタンが表示されます。

- 🔍+ / 🔍− : 拡大／縮小
- ⊞ : 画面に合わせてフィット
- ◀ ▶ ▲ ▼ : 上下左右にパン（移動）
- `Ctrl` を押しながらマウスホイール : 拡大縮小
- 図の上で**左ドラッグ**: パン（移動）

図がコンテナに収まりきらない場合は初回表示時に自動でフィットします。表示エリアの高さは画面の 70% までに制限されています。

**b. ダイアグラム編集画面（専用エディタ・簡易フォームエディタ共通）**

sequence / class / mindmap / quadrant / gantt / er の6種類の専用エディタと、その他の簡易フォームエディタでは、次のズーム／パン操作が使えます。

- 🔍+ / 🔍− : 拡大／縮小（% 表示付き）
- ⊞ : 画面に合わせてフィット
- ◀ ▶ ▲ ▼ : 上下左右にパン
- `Ctrl` を押しながらマウスホイール : 拡大縮小
- 図の上で**中ボタン（ホイールボタン）ドラッグ**: パン（移動）

**c. フローチャート編集画面のみ、パン非対応**

フローチャートのビジュアルエディタだけは独自実装のため、🔍+ / 🔍− / ⊞フィット と `Ctrl` +ホイールでの拡大縮小はできますが、**方向ボタンによるパンや中ボタンドラッグによるパンには対応していません**。図が大きい場合はズームアウトして全体を確認してください。

| `Escape` | 変更を破棄確認のうえ編集を完了（完了後、そのブロックにフォーカスが戻ります） |
| `Ctrl + Enter` | 変更を確定（同上、フォーカスが戻ります） |
| `Alt + ↑` / `Alt + ↓` | 確定して前後のブロックの編集へ直接ジャンプ |
| `Ctrl + B` | 太字の書式を適用 |
| `Ctrl + I` | 斜体の書式を適用 |
| `Tab` | 半角スペース4個を挿入 |

### 複数ブロック選択時（2件以上）

| キー | 動作 |
|---|---|
| `Ctrl + C` / `Ctrl + X` / `Ctrl + V` | コピー／切り取り／貼り付け |
| `Delete` / `Backspace` | 選択ブロックを削除（2件以上選択時のみ） |
| `Escape` | 選択解除 |

### グローバル

| キー | 動作 |
|---|---|
| `Ctrl + F` | 検索バーを表示 |
| `Ctrl + H` | 検索／置換バーを表示 |
| `Esc`（検索バー中） | 検索バーを閉じる |
| `Ctrl + Z` | 元に戻す（Undo） |
| `Ctrl + Shift + Z` / `Ctrl + Y` | やり直す（Redo） |

> `Ctrl + S` はこの拡張機能では処理されず、VS Code 標準の保存操作がそのまま使えます。

### Mermaid ／数式のコードエディタ内

| キー | 動作 |
|---|---|
| `Ctrl + Enter` | 変更を保存 |
| `Escape` | 編集をキャンセル（変更がある場合は確認） |
| `Tab` | 半角スペース4個を挿入 |

### フローチャートビジュアルエディタ内

| キー | 動作 |
|---|---|
| `Delete` / `Backspace` | 選択中のノードまたは接続を削除 |
| `Escape` | 接続モード終了 / 選択解除 |
| `Ctrl + Z` | 元に戻す |
| `Ctrl + Y` / `Ctrl + Shift + Z` | やり直す |
| `Enter`（テキスト編集中） | テキスト変更を確定 |

## 20. よくある質問（FAQ）

### Q1: 通常の Markdown エディタと併用できますか？

**A:** はい。VS Code の「Open With...」メニュー、または `.md` 編集中に表示される **「Markdown ビジュアルエディタで開く」ボタン**（タイトルバー）と **📝 テキストで開く ボタン**（ビジュアルエディタのツールバー）でいつでも切り替えられます。ビジュアルエディタで保存した内容は通常の Markdown ファイルとして使用できます。

### Q2: 既存の Markdown ファイルの構造は壊れませんか？

**A:** 変更していないブロックはそのまま保持されます。Mermaid ダイアグラムを保存する際はコードが再構成されますが、意味は変わりません。ただし、**テーブルの列配置指定（`:---:` など）は保存時に失われる**点にはご注意ください（[7. テーブルの編集](#7-テーブルの編集) 参照）。

### Q3: どのダイアグラムが GUI 編集に対応していますか？

**A:** フローチャート、クラス図、シーケンス図、マインドマップ、象限チャート、ガントチャート、ER 図の 7 種類が、ドラッグ操作を含む高機能な専用 GUI エディタに対応しています。その他の多くのダイアグラムも簡易フォーム形式の GUI で編集でき、ZenUML のみコード編集専用です。

### Q4: 画像はプレビューに表示されますか？

**A:** はい。Markdown 内の画像参照（`![](URL)`）はプレビュー表示されます。また、画像ファイルをドラッグ＆ドロップして挿入することもできます（[8. 画像の挿入](#8-画像の挿入) 参照）。

### Q5: VS Code Marketplace の拡張機能ですか？

**A:** いいえ。社内配布専用の拡張機能です。管理者から配布された VSIX ファイルからインストールしてください。

### Q6: アップデートはどうすればよいですか？

**A:** 管理者から新しい VSIX ファイルを受け取り、同じ手順でインストールしてください。既存バージョンをアンインストールする必要はありません。そのまま上書きインストールできます。

### Q7: リンクは何かキーを押さないと開けませんか？

**A:** いいえ。リンクは通常のクリックだけで開きます。修飾キーは不要です。

## 21. トラブルシューティング

### 「Markdown Visual Editor」が表示されない

- VS Code を完全に再起動（タスクバーのアイコンも閉じる）
- `Ctrl + Shift + P` → 「Developer: Reload Window」を実行
- インストール確認:
  ```powershell
  code --list-extensions | Select-String "md-visual-editor"
  ```

### Mermaid ダイアグラムが「レンダリングエラー」になる

- Mermaid コードが正しく &#96;&#96;&#96;mermaid ～ &#96;&#96;&#96; で囲まれているか確認
- コード先頭行が有効な宣言（`graph TD`、`sequenceDiagram` 等）で始まっているか確認
- VS Code のデベロッパーツール（`Ctrl + Shift + I`）でエラーを確認

### テーブル編集ボタンが表示されない

- Markdown テーブル記法が正しい形式か確認（ヘッダー行 + 区切り行 + データ行）

### テーブルの列配置（中央寄せなど）が消えてしまった

- 現在の仕様では、テーブルをビジュアルエディタで開いて保存すると、列の配置指定（`:---:` 等）は保持されません。配置を維持したい場合は、テキストエディタで直接編集してください。

### ドラッグ＆ドロップで画像が挿入されない

- ドロップ先が対応拡張子（`png`/`jpg`/`jpeg`/`gif`/`webp`/`svg`/`bmp`/`ico`/`avif`）かどうか確認
- `.md` ファイルが未保存の新規ファイルでないか確認（画像の保存先フォルダを決定するために、md ファイルの保存場所が必要です）

### VSIX のインストールに失敗する

- VS Code のバージョンが 1.80.0 以上か確認
- ファイルが破損していないか確認（再度コピー）
- 管理者権限は不要（通常ユーザーでインストール可能）
