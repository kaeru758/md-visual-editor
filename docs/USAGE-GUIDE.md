## 1. 前提条件

### 必須ソフトウェア

| ソフトウェア | バージョン | 確認コマンド |
|---|---|---|
| **Visual Studio Code** | 1.80.0 以上 | メニュー「ヘルプ」→「バージョン情報」 |
| **Node.js** | 18.x 以上（推奨: 18.17+） | `node --version` |
| **npm** | 8.x 以上 | `npm --version` |

### 推奨環境

- OS: Windows 10/11、macOS 12+、Ubuntu 20.04+
- メモリ: 4GB 以上
- ディスク空き容量: 500MB 以上（node_modules含む）

## 2. 環境構築手順

### 2.1 リポジトリのダウンロード/クローン

プロジェクトフォルダを任意の場所に配置します。

### 2.2 VS Code でプロジェクトを開く

```
1. VS Code を起動
2. メニュー「ファイル」→「フォルダーを開く」
3. プロジェクトフォルダ（拡張機能開発TRY）を選択
```

### 2.3 依存パッケージのインストール

VS Code 内のターミナル（`` Ctrl+` ``）で以下を実行：

```bash
npm install
```

> **所要時間:** 初回は約1〜3分（ネットワーク環境による）

### 2.4 ビルド

```bash
npm run compile
```

> 成功すると `dist/extension.js` が生成されます。

### 2.5 ビルド成功の確認

```bash
# Windows PowerShell
Test-Path dist\extension.js

# macOS / Linux
ls dist/extension.js
```

`True` または ファイルが表示されれば成功です。

## 3. 拡張機能の起動手順

### 3.1 デバッグモードで起動（推奨）

```
1. VS Code でプロジェクトフォルダを開いている状態で
2. キーボード「F5」を押す
   （または メニュー「実行」→「デバッグの開始」）
3. 新しい VS Code ウィンドウ（Extension Development Host）が開く
```

> **初回は自動的に `npm run watch` が実行されます。**  
> 「タスクを実行しますか？」と聞かれた場合は「はい」を選択してください。

### 3.2 起動確認

Extension Development Host ウィンドウが開いたら、拡張機能は正常にロードされています。

**確認方法：**
```
1. Extension Development Host で「Ctrl+Shift+P」（コマンドパレット）
2. 「Reopen With...」と入力
3. 候補に「Markdown Visual Editor」が表示されれば成功
```

### 3.3 ウォッチモードで開発する場合

ターミナルで以下を実行しておくと、ソースコード変更時に自動でリビルドされます：

```bash
npm run watch
```

ソース変更後は Extension Development Host で `Ctrl+R` を押すとリロードされます。

## 4. Markdownファイルをビジュアルエディタで開く

### 方法A: 右クリックメニューから開く（推奨）

```
1. エクスプローラーで .md ファイルを右クリック
2.「Open With...」（別のエディターで開く）を選択
3.「Markdown Visual Editor」を選択
```

### 方法B: 既に開いているファイルを切り替える

```
1. .md ファイルを通常のテキストエディタで開く
2. エディタタブの右上にある「…」メニューをクリック
3.「Reopen Editor With...」を選択
4.「Markdown Visual Editor」を選択
```

### 方法C: コマンドパレットから開く

```
1. .md ファイルを開いた状態で
2. Ctrl+Shift+P でコマンドパレットを開く
3.「Reopen With...」と入力
4.「Markdown Visual Editor」を選択
```

### 方法D: タイトルバーのボタン／コマンドから開く（v0.5.3 追加）

`.md` ファイルを標準のテキストエディタで開いているときは、エディタタブ右上（タイトルバー）に **「Markdown ビジュアルエディタで開く」** ボタン（プレビューアイコン）が表示されます。これをクリックするか、コマンドパレットで **「Markdown Visual Editor: Markdown ビジュアルエディタで開く」** を実行すると、ワンクリックでビジュアルエディタに切り替わります。

逆に、ビジュアルエディタから元のテキスト表示に戻したいときは、ツールバー右端の **📝 テキストで開く** ボタンを使います（[6. ツールバーの使い方](#6-ツールバーの使い方) 参照）。この2つのボタンで、テキスト編集とビジュアル編集を自由に行き来できます。

### テスト用ドキュメント

プロジェクトに含まれる `test-document.md` を上記の方法で開くと、各種Markdown要素とMermaidダイアグラムの動作を確認できます。

## 5. 基本操作 — ブロックの選択・編集・追加

ビジュアルエディタでは、Markdown の各要素が**レンダリングされた状態**で表示され、「ブロック」という単位で選択・編集します（v0.5.4 以降の仕様）。

### 5.1 ブロックの2種類

- **テキストセクション**: **H1（`#`）または H2（`##`）見出しから、次の H1/H2 見出しの直前まで**が1つのブロックになります。配下の H3〜H6・段落・リスト・引用・区切り線はすべて同じブロックに含まれます。見出しのない冒頭の文章も1つのセクションとして扱われます。
- **特殊ブロック**: テーブル、コードブロック（プレーンなコードに加え、Mermaid ダイアグラムや数式ブロックを含む）は、周囲のテキストと混ざらず**常に単独で1ブロック**になります。

> 例えば「見出し＋説明文」の下に表がある文書では、「見出し＋説明文」で1ブロック、「表」で別の1ブロックに分かれます。文章の一部を独立したブロックにしたい場合は、H1/H2 見出しを追加してセクションを分割してください。

### 5.2 ブロックの選択

- クリック → 単一選択
- `Ctrl + クリック` → 選択のオン/オフを切り替え（複数選択）
- `Shift + クリック` → 範囲選択
- 背景クリック → 選択解除

### 5.3 ブロックの編集手順

```
手順1: 編集したいブロックにマウスを移動
       → 青い枠線が表示され「ダブルクリックで編集」と表示される
       （ブロックを選択した状態で Enter / F2 でも編集開始できる）

手順2: ブロックをダブルクリック
       → テキストセクションの場合はテキストエリアに切り替わり、Markdownソースが表示される
       → テーブルの場合は各セルがその場で編集できるテキストボックスになる

手順3: テキストを編集
       → Markdownの記法で直接編集できる
       　 例: **太字** 、*斜体* 、- リスト項目

手順4: 編集完了
       → 以下のいずれかで確定：
         ・Escape キーを押す（変更があれば破棄確認）
         ・Ctrl+Enter を押す
         ・ブロックの外側をクリック
       → 編集内容が即座にレンダリングされ、ファイルに自動保存される
       → 確定後はそのブロックにフォーカス・選択が戻る（v0.5.3）
```

### 5.4 ブロックの追加（右クリックメニュー、v0.5.2 追加）

既存のブロックを右クリックし、**「⬆ 上にブロックを追加…」** または **「⬇ 下にブロックを追加…」** を選ぶと、サブメニューから挿入する要素の種類（段落 / H1〜H3 / 箇条書き / 番号付きリスト / タスクリスト / 表 / コードブロック / 数式 / 引用 / 水平線 / Mermaid ダイアグラム）を選べます。選択すると新しいブロックが挿入され、**そのまま編集状態に入ります**。

ブロックを選択している状態であれば、キーボードだけでも同じ操作が可能です。

| キー | 動作 |
|---|---|
| `Ctrl + Enter` | 「下にブロックを追加」メニューを開く |
| `Ctrl + Shift + Enter` | 「上にブロックを追加」メニューを開く |

### 5.5 キーボードだけで編集する（v0.5.3）

マウスなしでも一通りの編集が可能です。ブロックを選択した状態で `↑` / `↓` を押すと前後のブロックにフォーカスが移動し、`Enter` / `F2` で編集を開始できます。編集中に `Alt + ↑` / `Alt + ↓` を押すと、変更を確定してそのまま隣のブロックの編集に直接ジャンプできるため、複数ブロックを連続編集する作業がマウスなしで完結します。詳しい一覧は [12. キーボードショートカット一覧](#12-キーボードショートカット一覧) を参照してください。

### 5.6 テキスト選択して書式適用

```
手順1: ブロックをダブルクリックして編集モードに入る
手順2: 書式を適用したいテキストをマウスで選択
手順3: ツールバーの書式ボタン（B, I, S 等）をクリック
       → 選択テキストが Markdown記法で囲まれる
       　 例: 「テスト」を選択して B → 「**テスト**」
```

### 5.7 その他のブロック操作

ブロックを右クリックすると、選択状態に応じたメニューが表示されます。

- **単一選択時**: 編集 / 上下にブロックを追加 / 切り取り・コピー・貼り付け / 上下に移動 / PDF として出力 / 削除
- **複数選択時（2件以上）**: まとめて 切り取り・コピー・貼り付け・削除・PDF出力
- **背景を右クリック**: 末尾に貼り付け / PDF として出力

ブロックはドラッグ＆ドロップ（ハンドル `⋮⋮` またはブロック自体）でも並べ替えられます。削除時は確認ダイアログが表示され、削除前にクリップボードへ自動コピーされます。

## 6. ツールバーの使い方

画面上部のツールバーから、コードを書かずに各種要素を挿入できます。

### 6.1 書式ボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| **B** | 太字 | `**太字テキスト**` |
| *I* | 斜体 | `*斜体テキスト*` |
| ~~S~~ | 取り消し線 | `~~取り消しテキスト~~` |
| `</>` | インラインコード | `` `コード` `` |

### 6.2 見出しボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| H1 | 見出し1 | `# 見出し1` |
| H2 | 見出し2 | `## 見出し2` |
| H3 | 見出し3 | `### 見出し3` |

### 6.3 構造要素ボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| • List | 箇条書き | `- リスト項目` |
| 1. List | 番号付きリスト | `1. リスト項目` |
| 🔗 | リンク | `[リンクテキスト](URL)` |
| ⊞ | テーブル | 3列のテーブルテンプレート |

### 6.4 コード・図・数式系ボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| { } | コードブロック | ` ```コード``` ` |
| ∑ | 数式ブロック | ` ```math``` ` |
| ◇ Mermaid | Mermaidダイアグラム | ダイアグラム種別選択ピッカー（20 種類） |

### 6.5 ツールバー右端のボタン

| ボタン | 機能 |
|---|---|
| ●/○ 保存インジケータ | 未保存（dirty）状態の表示のみ（クリック不可） |
| ↶ / ↷ | Undo / Redo |
| 🔍 検索 | 検索／置換バーを表示（`Ctrl+F` / `Ctrl+H` でも起動可） |
| ☀️ / 🌙 テーマ切替 | エディタと Mermaid 図のテーマを 自動 → ライト強制 → ダーク強制 と循環（選択は保存される） |
| 📝 テキストで開く | 現在の Markdown を標準テキストエディタで開き直す（逆方向は [4. 方法D](#4-markdownファイルをビジュアルエディタで開く) 参照） |

### 6.6 ツールバーの動作ルール

- **編集中のブロックがある場合:** テキストエリアのカーソル位置に書式が挿入される
- **編集中のブロックがない場合:** **挿入位置ピッカー**が表示され、「先頭に挿入」または任意のブロックの後を選択できる

> Mermaid ボタンの場合は、先にダイアグラム種別（20種類）を選択し、その後挿入位置を指定します。

### 6.7 リンクのクリック動作

リンクは通常のクリックだけで開きます。**修飾キー（`Ctrl` / `Cmd`）は不要です。** `http(s)://` と `mailto:` は OS 標準のブラウザ／メーラーで、それ以外は VS Code 内で開きます。相対パスは `.md` ファイルの場所を基準に解決されます。

## 7. テーブルの編集

1. テーブルブロックを編集状態にすると、各セルが**自動で高さが伸び縮みするテキストボックス**になります。クリックして直接入力してください
2. セル内で改行すると Markdown の `<br>` に変換されて保存されます（`<br>` を含むセルを開くと、セル内では改行として表示されます）
3. セルを**右クリック**すると、列の追加（左/右）・行の追加（上/下）・この列を削除（2列以上のとき）・この行を削除（2行以上のとき）が行えます
4. 列の境界線をドラッグすると列幅を調整できますが、**これは表示上の調整のみで Markdown ファイルには保存されません**

> **重要な制限:** セルの配置（`:---:` などの左寄せ・中央・右寄せ指定）は GUI では設定できません。**既にセル配置が指定されている表をビジュアルエディタで開いて保存すると、その配置指定は失われます**（区切り行が `---` のみで書き出されるため）。配置を厳密に保ちたい表はテキストエディタ側で編集してください。テーブル編集に専用の Undo はありません。

## 8. 画像の挿入

画像ファイルを、**既存のブロックの上** または **エディタの余白** にドラッグ＆ドロップすると、Markdown の画像参照として自動挿入されます。

- 画像は開いている `.md` ファイルと同じ階層の **`images/` フォルダ**に保存されます（なければ自動作成、同名ファイルは自動リネーム）
- 対応拡張子: `png` / `jpg` / `jpeg` / `gif` / `webp` / `svg` / `bmp` / `ico` / `avif`
- 挿入される Markdown は `![alt](images/ファイル名)` の形式です
- `.md` が未保存の新規ファイルの場合、保存先フォルダを決定できないためドロップしても挿入できません。先に一度保存してください

## 9. 数式の編集

インライン数式（`$...$`）・ディスプレイ数式（`$$...$$`）・` ```math ` フェンスのブロック数式は、すべて KaTeX でレンダリングされます。

- ` ```math ` ブロックにマウスを移動すると表示される「✎ 数式を編集」ボタンから数式エディタが開きます
- 数式エディタは、LaTeX を入力するテキストエリアと、約200msのデバウンスで自動更新される**ライブプレビュー**で構成されます
- **LaTeX 記号パレット**（構造 / 演算子 / 関係 / 大記号 / ギリシャ文字 / 集合・論理 / 行列・整列 の7グループ）から、クリックだけでよく使う記号を挿入できます
- `Ctrl+Enter` で保存、`Escape` でキャンセル（変更があれば確認）

数式にエラーがある場合は赤みがかった背景で表示され、マウスを乗せるとエラー理由がツールチップで確認できます。

## 10. Mermaidダイアグラムの編集（ビジュアルエディタ）

フローチャート、クラス図、シーケンス図、マインドマップ、象限チャート、ガントチャート、ER 図は **ビジュアルエディタ（GUI）** で編集できます。その他の多くのMermaidダイアグラムも簡易フォーム形式のGUIで編集できますが（ZenUML のみコード編集専用）、本ガイドでは割愛します。

以下ではフローチャートのビジュアルエディタの詳細な操作方法を説明します。クラス図・シーケンス図・マインドマップ・象限チャート・ガントチャート・ER 図の操作方法は [USER-GUIDE.md](USER-GUIDE.md) を参照してください。

### 10.1 ビジュアルエディタの起動

```
手順1: Mermaidダイアグラムにマウスを移動
       → 右上に「✎ ダイアグラムを編集」ボタンが表示される

手順2: ボタンをクリック（またはダイアグラムをダブルクリック）
       → ビジュアルエディタが展開される

画面構成:
┌──────────────────────────────────────────────┐
│ [ノード追加ボタン群] [🔗接続] [↩戻す] [方向]  │ ← ツールバー
├──────────────────────────────────────────────┤
│ ヒント: ノードをクリックで選択...              │ ← ステータスバー
├──────────────────────────────────────────────┤
│                                              │
│         [ダイアグラム SVG 表示]                │ ← メインエリア
│         ノードをクリック/ダブルクリック操作      │
│                                              │
├──────────────────────────────────────────────┤
│ 接続一覧:  開始 → 処理  ✏️ ✕               │ ← 接続リスト
│           処理 → 終了  ✏️ ✕                │
├──────────────────────────────────────────────┤
│                   [キャンセル] [保存]          │ ← アクションバー
└──────────────────────────────────────────────┘
```

### 10.2 ノードの追加（コード不要）

```
手順1: ツールバーの形状ボタンをクリック
       □ = 矩形  ⬭ = 角丸  ◇ = 分岐(ひし形)  ○ = 円形
       ⬭ = スタジアム  ⬡ = 六角形  etc.

手順2: 新しいノード「新しいノード」が自動的に追加される
       → ダイアグラムに即座に反映

手順3: テキストを変更する場合はノードをダブルクリック（後述）
```

### 10.3 ノードのテキスト編集（コード不要）

```
手順1: ダイアグラム上のノードをダブルクリック
       → ノードの中央にテキスト入力欄が表示される

手順2: テキストを入力
       例: 「新しいノード」→「ユーザー認証」

手順3: 入力欄の下に形状切り替えバーが表示される
       → クリックするとノードの形状も同時に変更可能

手順4: Enter で確定 / Escape でキャンセル
```

### 10.4 ノードの形状変更（コード不要）

```
手順1: ノードをダブルクリック
手順2: テキスト入力欄の下に表示される形状バーで変更したい形状をクリック
       → 即座に形状が切り替わる
手順3: Enter で確定
```

### 10.5 ノード間の接続追加（コード不要）

```
手順1: ツールバーの「🔗 接続」ボタンをクリック
       → 接続モードに入る（ボタンがハイライトされる）
       → ステータスバーに「🔗 接続モード: 接続元のノードをクリックしてください」と表示

手順2: 接続元のノードをクリック
       → ノードが緑色にハイライト
       → 「🔗 接続元: 「○○」→ 接続先のノードをクリックしてください」と表示

手順3: 接続先のノードをクリック
       → ラベル入力ダイアログが表示される

手順4: ラベルを入力（空欄でもOK）→「追加」クリック
       → 接続線がダイアグラムに追加される

手順5: 続けて別の接続を追加するか、
       「🔗 接続」ボタンを再度クリックして接続モードを終了
```

### 10.6 接続線の種類を変更

```
手順1: ツールバーのドロップダウン（「→ 矢印」の右のセレクトボックス）で線種を選択
       ・→ 矢印（デフォルト）
       ・— 直線
       ・⋯ 点線
       ・⇒ 太線

手順2: 選択した線種は次に作成する接続に適用される
```

### 10.7 接続のラベル編集

```
方法A: 接続一覧パネルの ✏️ ボタンをクリック
       → ラベル入力ダイアログが表示される → 入力して確定

方法B: ダイアグラム上の接続線をダブルクリック
       → 操作パネルが表示される
       → ラベル編集 / 線種変更 / 方向反転 / 削除 を実行できる
```

サブグラフを SVG 上でダブルクリックすると、名前変更やノードの追加/除外も行えます。

### 10.8 要素の削除

```
方法A: 削除したいノード or 接続をクリックで選択 → Delete キー

方法B: ツールバーの「🗑 削除」ボタンをクリック

方法C: 接続一覧パネルの ✕ ボタンで接続を個別削除

※ ノードを削除すると、そのノードに接続されている接続線も自動的に削除されます
```

### 10.9 フロー方向の変更

```
手順: ツールバーの「方向」ドロップダウンで選択
     ・↓ 上から下（TD）— デフォルト
     ・→ 左から右（LR）
     ・↑ 下から上（BT）
     ・← 右から左（RL）
     → ダイアグラム全体のレイアウト方向が即座に変わる
```

### 10.10 元に戻す / やり直す

```
・ツールバーの「↩ 戻す」ボタン（または Ctrl+Z）
・ツールバーの「↪ やり直す」ボタン（または Ctrl+Y / Ctrl+Shift+Z）
```

### 10.11 ズーム／パン（v0.5.5）

```
・🔍+ / 🔍− / ⊞フィット、Ctrl+ホイールで拡大縮小
・※ フローチャートのビジュアルエディタは他のダイアグラムエディタと異なり、
  パン操作（方向ボタン・中ボタンドラッグ）には対応していません
```

他の専用ビジュアルエディタ（class / sequence / mindmap / quadrant / gantt / er）では、上記に加えて ◀▶▲▼ の方向ボタンと、図の上での**中ボタン（ホイールボタン）ドラッグ**によるパンも使えます。また、ドキュメント本文中のプレビューに表示される Mermaid 図は、専用エディタとは別の仕組みで拡大縮小・**左ドラッグでのパン**に対応しています。詳しくは [USER-GUIDE.md](USER-GUIDE.md) の「Mermaid 図の拡大・縮小」を参照してください。

### 10.12 コード表示 / コード直接編集

高度なユーザー向けに、Mermaidコードを直接表示・編集することも可能です。

```
手順1: ツールバーの「📝 コード」ボタンをクリック
       → 下部にMermaidコードのテキストエリアが表示される

手順2: コードを直接編集

手順3: 「コードを適用」ボタンをクリック
       → ダイアグラムに変更が反映される
```

### 10.13 編集の保存 / キャンセル

```
・「保存」ボタン → 変更をMarkdownファイルに反映してビジュアルエディタを閉じる
・「キャンセル」ボタン → 変更を破棄してビジュアルエディタを閉じる
```

## 11. Mermaidダイアグラムの編集（コードエディタ）

状態遷移図、パイチャート、Git グラフなど、専用ビジュアルエディタ非対応のMermaidダイアグラムの多くは**コード＋ライブプレビュー**の分割エディタで編集します（一部は簡易フォーム形式のGUIも利用できます。ZenUMLのみコード編集専用です）。

### 11.1 コードエディタの起動

```
手順: Mermaidダイアグラム（状態遷移図やパイチャート等）の「✎ ダイアグラムを編集」をクリック

画面構成:
┌──────────────────┬──────────────────┐
│  Mermaid コード    │  ライブプレビュー   │
│                  │                  │
│  stateDiagram-v2 │   ┌─────┐        │
│    [*] --> S1    │   │ S1  │──→     │
│    S1 --> [*]    │   └─────┘        │
│                  │                  │
├──────────────────┴──────────────────┤
│              [キャンセル] [保存]      │
└─────────────────────────────────────┘
```

### 11.2 操作方法

```
手順1: 左側のテキストエリアでMermaidコードを編集
       → 入力停止から約0.5秒後に右側のプレビューが自動更新

手順2: プレビューで結果を確認しながら調整

手順3: 「保存 (Ctrl+Enter)」で確定 / 「キャンセル」で破棄
```

### 11.3 対応するダイアグラム種類

| 種類 | エディタ | Mermaid先頭行 |
|---|---|---|
| フローチャート | **ビジュアルエディタ（GUI）** | `graph TD` / `flowchart LR` 等 |
| クラス図 | **ビジュアルエディタ（GUI）** | `classDiagram` |
| シーケンス図 | **ビジュアルエディタ（GUI）** | `sequenceDiagram` |
| マインドマップ | **ビジュアルエディタ（GUI）** | `mindmap` |
| 象限チャート | **ビジュアルエディタ（GUI）** | `quadrantChart` |
| ガントチャート | **ビジュアルエディタ（GUI）** | `gantt` |
| ER 図 | **ビジュアルエディタ（GUI）** | `erDiagram` |
| 状態遷移図 | コードエディタ（簡易フォームGUIあり） | `stateDiagram-v2` |
| パイチャート | コードエディタ（簡易フォームGUIあり） | `pie` |
| ZenUML | コードエディタのみ（挿入ピッカーからも除外） | `zenuml` |
| その他 | コードエディタ（多くは簡易フォームGUIあり） | （上記以外のMermaid構文） |

## 12. キーボードショートカット一覧

### ブロック選択中（編集していない）

| キー | 動作 |
|---|---|
| `Enter` / `F2` | 編集開始 |
| `Delete` | 削除（確認ダイアログあり） |
| `↑` / `↓` | 前後のブロックへフォーカス移動＋単一選択 |
| `Ctrl+Enter` | 「下にブロックを追加」メニューを開く |
| `Ctrl+Shift+Enter` | 「上にブロックを追加」メニューを開く |
| `Tab` | 次のブロックへフォーカス移動 |

### テキストセクション編集中（テキストエリア）

| キー | 動作 |
|---|---|
| `Escape` | 変更を破棄確認のうえ編集を完了（完了後、そのブロックにフォーカスが戻る） |
| `Ctrl+Enter` | 変更を確定（同上、フォーカスが戻る） |
| `Alt+↑` / `Alt+↓` | 確定して前後のブロックの編集へ直接ジャンプ |
| `Ctrl+B` | 太字の書式を適用 |
| `Ctrl+I` | 斜体の書式を適用 |
| `Tab` | 半角スペース4個の挿入 |

### 複数ブロック選択時（2件以上）

| キー | 動作 |
|---|---|
| `Ctrl+C` / `Ctrl+X` / `Ctrl+V` | コピー／切り取り／貼り付け |
| `Delete` / `Backspace` | 選択ブロックを削除（2件以上選択時のみ） |
| `Escape` | 選択解除 |

| `Ctrl+H` | 検索／置換バーを表示 |

## 13. よくある質問（FAQ）

### Q1: ビジュアルエディタは通常のMarkdownエディタと併用できますか？

**A:** はい。VS Codeの「Open With...」メニュー、`.md` 編集中に表示される **「Markdown ビジュアルエディタで開く」ボタン**（タイトルバー）、およびビジュアルエディタの **📝 テキストで開く** ボタンで、いつでも切り替えられます。ビジュアルエディタで保存した内容は通常のMarkdownファイルとしてそのまま使用できます。

### Q2: 既存のMarkdownファイルの構造は壊れませんか？

**A:** ブロック単位でRaw文字列を保持しているため、変更していないブロックはそのまま保存されます。ただし、Mermaidダイアグラムを保存する際はコードが再構成されます（意味は変わりませんがインデント等が正規化される場合があります）。また、**テーブルの列配置指定（`:---:` 等）はビジュアルエディタで保存すると失われる**点にご注意ください（[7. テーブルの編集](#7-テーブルの編集) 参照）。

### Q3: Mermaidビジュアルエディタでサブグラフは編集できますか？

**A:** フローチャートのビジュアルエディタでは「📦 グループ」ボタンからノードをサブグラフに追加できます。さらに、SVG 上でサブグラフをダブルクリックすると名前変更やノードの追加／除外ができます。「📝 コード」ボタンからコードを直接編集することも可能です。

### Q4: 画像はプレビューに表示されますか？

**A:** はい。Markdown内の画像参照（`![](URL)`）はプレビュー表示されます。ローカル相対パスの画像も表示されます。また、画像ファイルをブロックやエディタの余白にドラッグ＆ドロップして挿入することもできます（[8. 画像の挿入](#8-画像の挿入) 参照）。

### Q5: VS Code Marketplace に公開された拡張機能ですか？

**A:** いいえ。社内配布専用の拡張機能です。管理者から配布された VSIX ファイルからインストールしてください。インストール方法は [USER-GUIDE.md](USER-GUIDE.md) を参照してください。

### Q6: デフォルトのMarkdownエディタとして設定できますか？

**A:** `package.json` の `priority` を `"option"` から `"default"` に変更すると、`.md` ファイルを開くときにデフォルトでビジュアルエディタが使用されます。

### Q7: リンクは何かキーを押さないと開けませんか？

**A:** いいえ。リンクは通常のクリックだけで開きます。修飾キーは不要です。

## 14. トラブルシューティング

### 症状: F5を押してもExtension Development Hostが起動しない

**対処法:**
```
1. dist/extension.js が存在するか確認
   → なければ「npm run compile」を実行
2. node_modules が存在するか確認
   → なければ「npm install」を実行
3. .vscode/launch.json が正しいか確認
```

### 症状: 「Open With...」に「Markdown Visual Editor」が表示されない

**対処法:**
```
1. Extension Development Host ウィンドウで操作していることを確認
   （メインのVS Codeウィンドウでは拡張機能は有効になりません）
2. ファイルの拡張子が .md であることを確認
3. Extension Development Host を一度閉じて F5 で再起動
```

### 症状: Mermaidダイアグラムが「レンダリングエラー」になる

**対処法:**
```
1. Mermaidの構文が正しいか確認（https://mermaid.js.org/ の Live Editor で検証）
2. 「📝 コード」ボタンでコードを確認・修正
3. コード先頭行が「graph TD」「sequenceDiagram」等の有効な宣言で始まっているか確認
```

### 症状: 編集が保存されない

**対処法:**
```
1. Escape または Ctrl+Enter で編集を確定したか確認
   （ブロック外をクリックしても確定されます）
2. Mermaidビジュアルエディタの場合は「保存」ボタンをクリック
3. VS Code のファイルメニューから「保存」(Ctrl+S) を実行
```

### 症状: テーブルの列配置（中央寄せなど）が消えた

**対処法:**
```
現在の仕様では、テーブルをビジュアルエディタで開いて保存すると列の配置指定
（:---: 等）は保持されません。配置を維持したい場合はテキストエディタで
直接編集してください。
```

### 症状: 画像をドラッグ＆ドロップしても挿入されない

**対処法:**
```
1. 対応拡張子か確認（png/jpg/jpeg/gif/webp/svg/bmp/ico/avif）
2. .md ファイルが一度も保存されていない新規ファイルでないか確認
   （画像の保存先フォルダの決定に、md ファイルの保存場所が必要です）
```

### 症状: ビジュアルエディタの表示が崩れる

**対処法:**
```
1. Ctrl+Shift+P → 「Developer: Reload Window」を実行
2. それでも改善しない場合は Extension Development Host を再起動（F5）
```
