## 1. インストール方法

### 前提条件

- **VS Code** バージョン 1.80 以上
- **OS**: Windows 10 / 11

### 手順（VSIXファイルからインストール）

1. 共有フォルダから **`md-visual-editor-<バージョン>.vsix`** をローカルにコピー
   > 共有先パスは管理者から案内されます
2. VS Code を開く
3. `Ctrl + Shift + P` でコマンドパレットを開く
4. **「Extensions: Install from VSIX...」** と入力して選択
5. コピーした `.vsix` ファイルを選択
6. インストール完了後、**VS Code を再起動**

> コマンドラインからもインストールできます:
> ```powershell
> code --install-extension md-visual-editor-0.4.1.vsix
> ```

### インストール確認

1. VS Code で任意の `.md` ファイルを開く
2. ファイルを右クリック → **「エディターを開く方法」** → **「Markdown Visual Editor」** を選択
3. プレビュー形式の編集画面が表示されれば成功

## 2. Markdownファイルの開き方

### 方法A: 右クリックメニューから開く（推奨）

1. エクスプローラーで `.md` ファイルを右クリック
2. 「Open With...」（別のエディターで開く）を選択
3. 「Markdown Visual Editor」を選択

### 方法B: 既に開いているファイルを切り替える

1. `.md` ファイルを通常のテキストエディタで開く
2. エディタタブの右上にある「…」メニューをクリック
3. 「Reopen Editor With...」を選択
4. 「Markdown Visual Editor」を選択

### 方法C: コマンドパレットから開く

1. `.md` ファイルを開いた状態で `Ctrl + Shift + P`
2. 「Reopen With...」と入力
3. 「Markdown Visual Editor」を選択

## 3. 基本操作 — テキストブロックの編集

Markdown の各要素（見出し、段落、リスト、コードブロック等）が **レンダリングされた状態** で表示されます。

### 編集の流れ

1. **マウスを移動** → 青い枠線が表示され「ダブルクリックで編集」と表示
2. **ダブルクリック** → テキストエリアに切り替わり、Markdown ソースが表示
3. **テキストを編集** → Markdown 記法で直接編集（`**太字**`、`*斜体*`、`- リスト`等）
4. **確定** → 以下のいずれかで確定:
   - `Escape` キー
   - `Ctrl + Enter`
   - ブロックの外側をクリック

編集内容は即座にレンダリングされ、ファイルに自動保存されます。

## 4. ツールバーの使い方

画面上部のツールバーから各種要素を挿入できます。

### 書式ボタン

| ボタン | 機能 | 挿入される Markdown |
|---|---|---|
| **B** | 太字 | `**太字テキスト**` |
| *I* | 斜体 | `*斜体テキスト*` |
| ~~S~~ | 取り消し線 | `~~取り消しテキスト~~` |
| `</>` | インラインコード | `` `コード` `` |

### 見出しボタン

| ボタン | 機能 |
|---|---|
| H1 | `# 見出し1` を挿入 |
| H2 | `## 見出し2` を挿入 |
| H3 | `### 見出し3` を挿入 |

### 構造要素ボタン

| ボタン | 機能 |
|---|---|
| • List | 箇条書きリストを挿入 |
| 1. List | 番号付きリストを挿入 |
| 🔗 | リンクを挿入 |
| ⊞ | テーブルテンプレートを挿入 |
| { } | コードブロックを挿入 |
| ◇ Mermaid | ダイアグラム種別選択ピッカーを表示（21 種類から選択） |

### ツールバー右端のボタン（v0.3.1 追加）

| ボタン | 機能 |
|---|---|
| 🔍 検索 | **検索／置換バー**を表示。`Ctrl + F` / `Ctrl + H` でも起動でき、大文字区別・正規表現・一括置換に対応 |
| ☀️ / 🌙 テーマ切替 | エディタと Mermaid 図のテーマを **自動（VS Code に追従） → ライト強制 → ダーク強制** と循環。選択は保存される |
| 📝 テキストで開く | 現在の Markdown を **標準テキストエディタ** で開き直し |

### 動作ルール

- **編集中のブロックがある場合**: カーソル位置に挿入
- **編集中のブロックがない場合**: **挿入位置ピッカー**が表示され、「先頭に挿入」または任意のブロックの後に挿入位置を選択できます

> **Mermaid ボタンの場合:** まずダイアグラムの種類（21 種類から選択）を選んだ上で、挿入位置ピッカーで位置を指定します。

### リンクのクリック動作（v0.3.1 追加）

- 通常のクリックではリンクを踏まず、誤作動を防ぐようになりました。
- リンクを開くときは **`Ctrl + Click`**（macOS は `Cmd + Click`）を使います。
  - `http://` / `https://` / `mailto:` は OS のデフォルトブラウザ／メーラーで開きます。
  - その他（相対パス・`file:` など）は VS Code 内で開きます。

## 5. テーブルの編集

Markdown のテーブルは GUI で編集できます。

### 操作手順

1. テーブルにマウスを移動 → 「✎ テーブルを編集」ボタンが表示
2. ボタンをクリック → GUI 編集モードに切り替え
3. セルをクリックして直接入力
4. 「➕ 列追加」「➕ 行追加」ボタンで拡張
5. 各列・行の「✕」ボタンで削除
6. 「保存」ボタンで確定 / 「キャンセル」で破棄

## 6. Mermaidダイアグラムの編集

Mermaid コードブロック（` ```mermaid ` ～ ` ``` `）にマウスを移動すると「✎ ダイアグラムを編集」ボタンが表示されます。

ダイアグラムの種類に応じて、**ビジュアルエディタ（GUI）** または **コードエディタ** が起動します。

### 対応一覧

| 種類 | エディタ | Mermaid 先頭行 |
|---|---|---|
| フローチャート | **ビジュアル（GUI）** | `graph TD` / `flowchart LR` 等 |
| クラス図 | **ビジュアル（GUI）** | `classDiagram` |
| シーケンス図 | **ビジュアル（GUI）** | `sequenceDiagram` |
| マインドマップ | **ビジュアル（GUI）** | `mindmap` |
| 象限チャート | **ビジュアル（GUI）** | `quadrantChart` |
| ガントチャート | **ビジュアル（GUI）** | `gantt` |
| ER 図 | **ビジュアル（GUI）** | `erDiagram` |
| その他すべて | コード + プレビュー | 上記以外の Mermaid 構文 |

## 7. フローチャート（ビジュアルエディタ）

draw.io 風の GUI でフローチャートを編集できます。コードを書く必要はありません。

### 画面構成

```
┌──────────────────────────────────────────────┐
│ [ノード追加ボタン群] [🔗接続] [↩戻す] [方向]  │ ← ツールバー
├──────────────────────────────────────────────┤
│ ヒント: ノードをクリックで選択...              │ ← ステータスバー
├──────────────────────────────────────────────┤
│          [ダイアグラム SVG 表示]               │ ← メインキャンバス
├──────────────────────────────────────────────┤
│ 接続一覧:  開始 → 処理  ✏️ ✕               │ ← 接続リスト
├──────────────────────────────────────────────┤
│                   [キャンセル] [保存]          │ ← アクションバー
└──────────────────────────────────────────────┘
```

### 主な操作

| 操作 | 方法 |
|---|---|
| ノード追加 | ツールバーの形状ボタン（□ ⬭ ◇ ○ 等）をクリック |
| テキスト編集 | ノードをダブルクリック → テキスト入力 → Enter で確定 |
| 形状変更 | ダブルクリック後に表示される形状バーで選択 |
| 色変更 | ノードを選択 → カラーボタンで塗り色・テキスト色を変更 |
| 接続追加 | 「🔗 接続」→ 接続元クリック → 接続先クリック → ラベル入力 |
| 線種変更 | ツールバーのドロップダウンで選択（矢印・直線・点線・太線） |
| 接続線の詳細編集 | SVG 上の接続線をダブルクリック → 方向反転・削除・線種変更・ラベル編集 |
| 方向変更 | 「方向」ドロップダウンで TD / LR / BT / RL を選択 |
| グループ化 | ノードを選択 → 「📦 グループ」ボタンでサブグラフに追加 |
| サブグラフ編集 | SVG 上のサブグラフをダブルクリック → 名前変更・ノード追加/除外 |
| 削除 | ノードまたは接続を選択 → Delete キーまたは「🗑 削除」ボタン |
| 元に戻す | Ctrl + Z / 「↩ 戻す」ボタン |
| やり直す | Ctrl + Y / Ctrl + Shift + Z |

```
┌─────────────────────────────────────────────────┐
│ ツールバー                                        │
│ [👤参加者追加][💬メッセージ追加][📝ノート追加]        │

├─────────────────────────────────────────────────┤
│ ▼ リスト（折りたたみ可能）                         │
│  参加者: 👤 User1 (表示名)  [✏️][✕]             │
│  メッセージ: 💬 User1 → User2: msg  [✏️][✕]     │
│              🟦 ブロック開始 [色]    [✏️][✕]      │
├─────────────────────────────────────────────────┤
│                   [キャンセル] [保存]               │

### 主な操作

| 操作 | 方法 |

| ブロック削除 | リストの「🟦 ブロック開始」行の ✕ ボタン |
| 順番変更 | リスト内のドラッグ＆ドロップで並び替え |
| SVGドラッグ | 参加者間をドラッグしてメッセージを追加 |
| SVGクリック | メッセージの線をクリック → リストでハイライト |
| SVGダブルクリック | メッセージのテキストをダブルクリック → テキスト編集 |
| 削除 | 各項目の「✕」ボタン |

## 10. マインドマップ（ビジュアルエディタ）

ツリー構造のマインドマップを GUI で編集できます。

### 画面構成

```
┌──────────────┬──────────────────┐
│  左パネル     │  プレビュー       │
│  ・ツリー表示  │  ・マインドマップ  │
│   ├ 子ノード   │    SVG表示       │

├──────────────┴──────────────────┤
│           [キャンセル] [保存]      │
└─────────────────────────────────┘
```

### 主な操作

| 操作 | 方法 |
|---|---|
| 子ノード追加 | 各ノードの「+」ボタン |
| テキスト編集 | 「✎」ボタン → テキスト入力 → 確定 |

| 操作 | 方法 |

## 14. その他のMermaidダイアグラム（コードエディタ）

上記の GUI エディタに対応していないダイアグラム（状態遷移図、パイチャート等）は、**コード + ライブプレビュー** の分割エディタで編集します。

### 画面構成

```
┌──────────────────┬──────────────────┐
│  Mermaid コード    │  ライブプレビュー   │
│                  │                  │
│  stateDiagram-v2 │   [SVG表示]       │
│    [*] --> s1    │                  │
│    s1 --> [*]    │                  │
├──────────────────┴──────────────────┤
│              [キャンセル] [保存]       │
└──────────────────────────────────────┘
```

### 操作方法

1. 左側のテキストエリアで Mermaid コードを編集
2. 入力停止から約 0.5 秒後にプレビューが自動更新
3. 「保存 (Ctrl+Enter)」で確定 / 「キャンセル」で破棄

## 16. よくある質問（FAQ）

### Q1: 通常の Markdown エディタと併用できますか？

**A:** はい。VS Code の「Open With...」メニューでいつでも切り替えられます。ビジュアルエディタで保存した内容は通常の Markdown ファイルとして使用できます。

### Q2: 既存の Markdown ファイルの構造は壊れませんか？

**A:** 変更していないブロックはそのまま保持されます。Mermaid ダイアグラムを保存する際はコードが再構成されますが、意味は変わりません。

### Q3: どのダイアグラムが GUI 編集に対応していますか？

**A:** フローチャート、クラス図、シーケンス図、マインドマップ、象限チャート、ガントチャート、ER 図の 7 種類が GUI 編集に対応しています。その他のダイアグラムはコード + プレビューの分割エディタで編集します。

### Q4: 画像はプレビューに表示されますか？

**A:** はい。Markdown 内の画像参照（`![](URL)`）はプレビュー表示されます。

### Q5: VS Code Marketplace の拡張機能ですか？

**A:** いいえ。社内配布専用の拡張機能です。管理者から配布された VSIX ファイルからインストールしてください。

### Q6: アップデートはどうすればよいですか？

**A:** 管理者から新しい VSIX ファイルを受け取り、同じ手順でインストールしてください。既存バージョンをアンインストールする必要はありません。そのまま上書きインストールできます。

## 17. トラブルシューティング

### 「Markdown Visual Editor」が表示されない

- VS Code を完全に再起動（タスクバーのアイコンも閉じる）
- `Ctrl + Shift + P` → 「Developer: Reload Window」を実行
- インストール確認:
  ```powershell
  code --list-extensions | Select-String "md-visual-editor"
  ```

### Mermaid ダイアグラムが「レンダリングエラー」になる

- Mermaid コードが正しく ` ```mermaid ` ～ ` ``` ` で囲まれているか確認
- コード先頭行が有効な宣言（`graph TD`、`sequenceDiagram` 等）で始まっているか確認
- VS Code のデベロッパーツール（`Ctrl + Shift + I`）でエラーを確認

### テーブル編集ボタンが表示されない

- Markdown テーブル記法が正しい形式か確認（ヘッダー行 + 区切り行 + データ行）

### VSIX のインストールに失敗する

- VS Code のバージョンが 1.80.0 以上か確認
- ファイルが破損していないか確認（再度コピー）
- 管理者権限は不要（通常ユーザーでインストール可能）
