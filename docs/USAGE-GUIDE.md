## 1. 前提条件

### 必須ソフトウェア

| ソフトウェア | バージョン | 確認コマンド |
|---|---|---|
| **Visual Studio Code** | 1.80.0 以上 | メニュー「ヘルプ」→「バージョン情報」 |
| **Node.js** | 18.x 以上（推奨: 18.17+） | `node --version` |
| **npm** | 8.x 以上 | `npm --version` |

### 推奨環境

- OS: Windows 10/11、macOS 12+、Ubuntu 20.04+
- メモリ: 4GB 以上
- ディスク空き容量: 500MB 以上（node_modules含む）

## 2. 環境構築手順

### 2.1 リポジトリのダウンロード/クローン

プロジェクトフォルダを任意の場所に配置します。

### 2.2 VS Code でプロジェクトを開く

```
1. VS Code を起動
2. メニュー「ファイル」→「フォルダーを開く」
3. プロジェクトフォルダ（拡張機能開発TRY）を選択
```

### 2.3 依存パッケージのインストール

VS Code 内のターミナル（`` Ctrl+` ``）で以下を実行：

```bash
npm install
```

> **所要時間:** 初回は約1〜3分（ネットワーク環境による）

### 2.4 ビルド

```bash
npm run compile
```

> 成功すると `dist/extension.js` が生成されます。

### 2.5 ビルド成功の確認

```bash
# Windows PowerShell
Test-Path dist\extension.js

# macOS / Linux
ls dist/extension.js
```

`True` または ファイルが表示されれば成功です。

## 3. 拡張機能の起動手順

### 3.1 デバッグモードで起動（推奨）

```
1. VS Code でプロジェクトフォルダを開いている状態で
2. キーボード「F5」を押す
   （または メニュー「実行」→「デバッグの開始」）
3. 新しい VS Code ウィンドウ（Extension Development Host）が開く
```

> **初回は自動的に `npm run watch` が実行されます。**  
> 「タスクを実行しますか？」と聞かれた場合は「はい」を選択してください。

### 3.2 起動確認

Extension Development Host ウィンドウが開いたら、拡張機能は正常にロードされています。

**確認方法：**
```
1. Extension Development Host で「Ctrl+Shift+P」（コマンドパレット）
2. 「Reopen With...」と入力
3. 候補に「Markdown Visual Editor」が表示されれば成功
```

### 3.3 ウォッチモードで開発する場合

ターミナルで以下を実行しておくと、ソースコード変更時に自動でリビルドされます：

```bash
npm run watch
```

ソース変更後は Extension Development Host で `Ctrl+R` を押すとリロードされます。

## 4. Markdownファイルをビジュアルエディタで開く

### 方法A: 右クリックメニューから開く（推奨）

```
1. エクスプローラーで .md ファイルを右クリック
2.「Open With...」（別のエディターで開く）を選択
3.「Markdown Visual Editor」を選択
```

### 方法B: 既に開いているファイルを切り替える

```
1. .md ファイルを通常のテキストエディタで開く
2. エディタタブの右上にある「…」メニューをクリック
3.「Reopen Editor With...」を選択
4.「Markdown Visual Editor」を選択
```

### 方法C: コマンドパレットから開く

```
1. .md ファイルを開いた状態で
2. Ctrl+Shift+P でコマンドパレットを開く
3.「Reopen With...」と入力
4.「Markdown Visual Editor」を選択
```

### テスト用ドキュメント

プロジェクトに含まれる `test-document.md` を上記の方法で開くと、各種Markdown要素とMermaidダイアグラムの動作を確認できます。

## 5. 基本操作 — テキストブロックの編集

ビジュアルエディタでは、Markdownの各要素（見出し、段落、リスト、テーブル、コードブロック等）が**レンダリングされた状態で表示**されます。

### 5.1 ブロックの編集手順

```
手順1: 編集したいブロック（見出し、段落、リスト等）にマウスを移動
       → 青い枠線が表示され「ダブルクリックで編集」と表示される

手順2: ブロックをダブルクリック
       → テキストエリアに切り替わり、Markdownソースが表示される

手順3: テキストを編集
       → Markdownの記法で直接編集できる
       　 例: **太字** 、*斜体* 、- リスト項目

手順4: 編集完了
       → 以下のいずれかで確定：
         ・Escape キーを押す
         ・Ctrl+Enter を押す
         ・ブロックの外側をクリック
       → 編集内容が即座にレンダリングされ、ファイルに自動保存される
```

### 5.2 編集中に使えるショートカット

| 操作 | ショートカット | 結果 |
|---|---|---|
| 太字 | `Ctrl+B` | カーソル位置に `**太字テキスト**` を挿入 |
| 斜体 | `Ctrl+I` | カーソル位置に `*斜体テキスト*` を挿入 |
| 編集確定 | `Escape` または `Ctrl+Enter` | 変更を保存してプレビュー表示に戻る |
| タブ入力 | `Tab` | 4スペースのインデントを挿入 |

### 5.3 テキスト選択して書式適用

```
手順1: ブロックをダブルクリックして編集モードに入る
手順2: 書式を適用したいテキストをマウスで選択
手順3: ツールバーの書式ボタン（B, I, S 等）をクリック
       → 選択テキストが Markdown記法で囲まれる
       　 例: 「テスト」を選択して B → 「**テスト**」
```

## 6. ツールバーの使い方

画面上部のツールバーから、コードを書かずに各種要素を挿入できます。

### 6.1 書式ボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| **B** | 太字 | `**太字テキスト**` |
| *I* | 斜体 | `*斜体テキスト*` |
| ~~S~~ | 取り消し線 | `~~取り消しテキスト~~` |
| `</>` | インラインコード | `` `コード` `` |

### 6.2 見出しボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| H1 | 見出し1 | `# 見出し1` |
| H2 | 見出し2 | `## 見出し2` |
| H3 | 見出し3 | `### 見出し3` |

### 6.3 構造要素ボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| • List | 箇条書き | `- リスト項目` |
| 1. List | 番号付きリスト | `1. リスト項目` |
| 🔗 | リンク | `[リンクテキスト](URL)` |
| ⊞ | テーブル | 3列のテーブルテンプレート |

### 6.4 コード系ボタン

| ボタン | 機能 | 挿入されるMarkdown |
|---|---|---|
| { } | コードブロック | ` ```コード``` ` |
| ◇ Mermaid | Mermaidダイアグラム | ダイアグラム種別選択ピッカー（21 種類 / v0.3.0+） |

### 6.5 ツールバーの動作ルール

- **編集中のブロックがある場合:** テキストエリアのカーソル位置に書式が挿入される
- **編集中のブロックがない場合:** **挿入位置ピッカー**が表示され、「先頭に挿入」または任意のブロックの後を選択できる

> Mermaid ボタンの場合は、先にダイアグラム種別を選択し、その後挿入位置を指定します。

## 7. Mermaidダイアグラムの編集（ビジュアルエディタ）

フローチャート、クラス図、シーケンス図、マインドマップ、象限チャート、ガントチャート、ER 図は **ビジュアルエディタ（GUI）** で編集できます。その他のMermaidダイアグラムはコード＋ライブプレビューのエディタを使用します。

以下ではフローチャートのビジュアルエディタの詳細な操作方法を説明します。クラス図・シーケンス図・マインドマップ・象限チャート・ガントチャートの操作方法は [USER-GUIDE.md](USER-GUIDE.md) を参照してください。

### 7.1 ビジュアルエディタの起動

```
手順1: Mermaidダイアグラムにマウスを移動
       → 右上に「✎ ダイアグラムを編集」ボタンが表示される

手順2: ボタンをクリック（またはダイアグラムをダブルクリック）
       → ビジュアルエディタが展開される

画面構成:
┌──────────────────────────────────────────────┐
│ [ノード追加ボタン群] [🔗接続] [↩戻す] [方向]  │ ← ツールバー
├──────────────────────────────────────────────┤
│ ヒント: ノードをクリックで選択...              │ ← ステータスバー
├──────────────────────────────────────────────┤
│                                              │
│         [ダイアグラム SVG 表示]                │ ← メインエリア
│         ノードをクリック/ダブルクリック操作      │
│                                              │
├──────────────────────────────────────────────┤
│ 接続一覧:  開始 → 処理  ✏️ ✕               │ ← 接続リスト
│           処理 → 終了  ✏️ ✕                │
├──────────────────────────────────────────────┤
│                   [キャンセル] [保存]          │ ← アクションバー
└──────────────────────────────────────────────┘
```

### 7.2 ノードの追加（コード不要）

```
手順1: ツールバーの形状ボタンをクリック
       □ = 矩形  ⬭ = 角丸  ◇ = 分岐(ひし形)  ○ = 円形
       ⬭ = スタジアム  ⬡ = 六角形  etc.

手順2: 新しいノード「新しいノード」が自動的に追加される
       → ダイアグラムに即座に反映

手順3: テキストを変更する場合はノードをダブルクリック（後述）
```

### 7.3 ノードのテキスト編集（コード不要）

```
手順1: ダイアグラム上のノードをダブルクリック
       → ノードの中央にテキスト入力欄が表示される

手順2: テキストを入力
       例: 「新しいノード」→「ユーザー認証」

手順3: 入力欄の下に形状切り替えバーが表示される
       → クリックするとノードの形状も同時に変更可能

手順4: Enter で確定 / Escape でキャンセル
```

### 7.4 ノードの形状変更（コード不要）

```
手順1: ノードをダブルクリック
手順2: テキスト入力欄の下に表示される形状バーで変更したい形状をクリック
       → 即座に形状が切り替わる
手順3: Enter で確定
```

### 7.5 ノード間の接続追加（コード不要）

```
手順1: ツールバーの「🔗 接続」ボタンをクリック
       → 接続モードに入る（ボタンがハイライトされる）
       → ステータスバーに「🔗 接続モード: 接続元のノードをクリックしてください」と表示

手順2: 接続元のノードをクリック
       → ノードが緑色にハイライト
       → 「🔗 接続元: 「○○」→ 接続先のノードをクリックしてください」と表示

手順3: 接続先のノードをクリック
       → ラベル入力ダイアログが表示される

手順4: ラベルを入力（空欄でもOK）→「追加」クリック
       → 接続線がダイアグラムに追加される

手順5: 続けて別の接続を追加するか、
       「🔗 接続」ボタンを再度クリックして接続モードを終了
```

### 7.6 接続線の種類を変更

```
手順1: ツールバーのドロップダウン（「→ 矢印」の右のセレクトボックス）で線種を選択
       ・→ 矢印（デフォルト）
       ・— 直線
       ・⋯ 点線
       ・⇒ 太線

手順2: 選択した線種は次に作成する接続に適用される
```

### 7.7 接続のラベル編集

```
方法A: 接続一覧パネルの ✏️ ボタンをクリック
       → ラベル入力ダイアログが表示される → 入力して確定

方法B: ダイアグラム上の接続線をダブルクリック
       → 操作パネルが表示される
       → ラベル編集 / 線種変更 / 方向反転 / 削除 を実行できる
```

サブグラフを SVG 上でダブルクリックすると、名前変更やノードの追加/除外も行えます。

### 7.8 要素の削除

```
方法A: 削除したいノード or 接続をクリックで選択 → Delete キー

方法B: ツールバーの「🗑 削除」ボタンをクリック

方法C: 接続一覧パネルの ✕ ボタンで接続を個別削除

※ ノードを削除すると、そのノードに接続されている接続線も自動的に削除されます
```

### 7.9 フロー方向の変更

```
手順: ツールバーの「方向」ドロップダウンで選択
     ・↓ 上から下（TD）— デフォルト
     ・→ 左から右（LR）
     ・↑ 下から上（BT）
     ・← 右から左（RL）
     → ダイアグラム全体のレイアウト方向が即座に変わる
```

### 7.10 元に戻す / やり直す

```
・ツールバーの「↩ 戻す」ボタン（または Ctrl+Z）
・ツールバーの「↪ やり直す」ボタン（または Ctrl+Y / Ctrl+Shift+Z）
```

### 7.11 コード表示 / コード直接編集

高度なユーザー向けに、Mermaidコードを直接表示・編集することも可能です。

```
手順1: ツールバーの「📝 コード」ボタンをクリック
       → 下部にMermaidコードのテキストエリアが表示される

手順2: コードを直接編集

手順3: 「コードを適用」ボタンをクリック
       → ダイアグラムに変更が反映される
```

### 7.12 編集の保存 / キャンセル

```
・「保存」ボタン → 変更をMarkdownファイルに反映してビジュアルエディタを閉じる
・「キャンセル」ボタン → 変更を破棄してビジュアルエディタを閉じる
```

## 8. Mermaidダイアグラムの編集（コードエディタ）

状態遷移図、パイチャート、Git グラフなど、ビジュアルエディタ非対応のMermaidダイアグラムは**コード＋ライブプレビュー**の分割エディタで編集します。

### 8.1 コードエディタの起動

```
手順: Mermaidダイアグラム（状態遷移図やパイチャート等）の「✎ ダイアグラムを編集」をクリック

画面構成:
┌──────────────────┬──────────────────┐
│  Mermaid コード    │  ライブプレビュー   │
│                  │                  │
│  stateDiagram-v2 │   ┌─────┐        │
│    [*] --> S1    │   │ S1  │──→     │
│    S1 --> [*]    │   └─────┘        │
│                  │                  │
├──────────────────┴──────────────────┤
│              [キャンセル] [保存]      │
└─────────────────────────────────────┘
```

### 8.2 操作方法

```
手順1: 左側のテキストエリアでMermaidコードを編集
       → 入力停止から約0.5秒後に右側のプレビューが自動更新

手順2: プレビューで結果を確認しながら調整

手順3: 「保存 (Ctrl+Enter)」で確定 / 「キャンセル」で破棄
```

### 8.3 対応するダイアグラム種類

| 種類 | エディタ | Mermaid先頭行 |

| フローチャート | **ビジュアルエディタ（GUI）** | `graph TD` / `flowchart LR` 等 |

| シーケンス図 | **ビジュアルエディタ（GUI）** | `sequenceDiagram` |
| マインドマップ | **ビジュアルエディタ（GUI）** | `mindmap` |
| 象限チャート | **ビジュアルエディタ（GUI）** | `quadrantChart` |
| ガントチャート | **ビジュアルエディタ（GUI）** | `gantt` |
| ER 図 | **ビジュアルエディタ（GUI）** | `erDiagram` |
| 状態遷移図 | コードエディタ | `stateDiagram-v2` |
| パイチャート | コードエディタ | `pie` |
| その他すべて | コードエディタ | （上記以外のMermaid構文） |

## 10. よくある質問（FAQ）

### Q1: ビジュアルエディタは通常のMarkdownエディタと併用できますか？

**A:** はい。VS Codeの「Open With...」メニューでいつでも切り替えられます。同じファイルを通常のテキストエディタで開き直すことも可能です。ビジュアルエディタで保存した内容は通常のMarkdownファイルとしてそのまま使用できます。

### Q2: 既存のMarkdownファイルの構造は壊れませんか？

**A:** MarkdownパーサーによるトークンベースのブロックレベルでRaw文字列を保持しているため、変更していないブロックはそのまま保存されます。ただし、Mermaidダイアグラムを保存する際はコードが再構成されます（意味は変わりませんがインデント等が正規化される場合があります）。

### Q3: Mermaidビジュアルエディタでサブグラフは編集できますか？

**A:** フローチャートのビジュアルエディタでは「📦 グループ」ボタンからノードをサブグラフに追加できます。さらに、SVG 上でサブグラフをダブルクリックすると名前変更やノードの追加／除外ができます。「📝 コード」ボタンからコードを直接編集することも可能です。

### Q4: 画像はプレビューに表示されますか？

**A:** はい。Markdown内の画像参照（`![](URL)`）はプレビュー表示されます。ローカル相対パスの画像も表示されます。

### Q5: VS Code Marketplace に公開された拡張機能ですか？

**A:** いいえ。社内配布専用の拡張機能です。管理者から配布された VSIX ファイルからインストールしてください。インストール方法は [USER-GUIDE.md](USER-GUIDE.md) を参照してください。

### Q6: デフォルトのMarkdownエディタとして設定できますか？

**A:** `package.json` の `priority` を `"option"` から `"default"` に変更すると、`.md` ファイルを開くときにデフォルトでビジュアルエディタが使用されます。

## 11. トラブルシューティング

### 症状: F5を押してもExtension Development Hostが起動しない

**対処法:**
```
1. dist/extension.js が存在するか確認
   → なければ「npm run compile」を実行
2. node_modules が存在するか確認
   → なければ「npm install」を実行
3. .vscode/launch.json が正しいか確認
```

### 症状: 「Open With...」に「Markdown Visual Editor」が表示されない

**対処法:**
```
1. Extension Development Host ウィンドウで操作していることを確認
   （メインのVS Codeウィンドウでは拡張機能は有効になりません）
2. ファイルの拡張子が .md であることを確認
3. Extension Development Host を一度閉じて F5 で再起動
```

### 症状: Mermaidダイアグラムが「レンダリングエラー」になる

**対処法:**
```
1. Mermaidの構文が正しいか確認（https://mermaid.js.org/ の Live Editor で検証）
2. 「📝 コード」ボタンでコードを確認・修正
3. コード先頭行が「graph TD」「sequenceDiagram」等の有効な宣言で始まっているか確認
```

### 症状: 編集が保存されない

**対処法:**
```
1. Escape または Ctrl+Enter で編集を確定したか確認
   （ブロック外をクリックしても確定されます）
2. Mermaidビジュアルエディタの場合は「保存」ボタンをクリック
3. VS Code のファイルメニューから「保存」(Ctrl+S) を実行
```

### 症状: ビジュアルエディタの表示が崩れる

**対処法:**
```
1. Ctrl+Shift+P → 「Developer: Reload Window」を実行
2. それでも改善しない場合は Extension Development Host を再起動（F5）
```
